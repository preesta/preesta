# Cookbook

Realistic rules you can copy and adjust. Each recipe is one rule (or a small set) solving a concrete problem; the title says what the rule answers.

## By problem

| I want… | Recipe |
|---|---|
| A morning digest of what I have to do today | [Daily standup](daily-standup.md) |
| A periodic ping about stale PRs/MRs | [Stale PRs](stale-prs.md) |
| To catch tickets that should already be closed | [Overdue tickets](overdue-tickets.md) |
| To automatically comment on neglected items | [Auto-comment stale](auto-comment-stale.md) |
| One digest mixing Jira + Linear + GitHub | [Multi-tracker digest](multi-tracker-digest.md) |
| Different rules for different teams | [Per-team routing](per-team-routing.md) |

## Reading order for newcomers

1. [Daily standup](daily-standup.md) — exercises basic shape: filter + recipient routing
2. [Per-team routing](per-team-routing.md) — shows the marker-based fan-out
3. [Auto-comment stale](auto-comment-stale.md) — adds the mutation side
4. [Multi-tracker digest](multi-tracker-digest.md) — combines all three pieces
